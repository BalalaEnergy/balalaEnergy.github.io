更改ssh的连接端口
2021年05月24日 172 字 大概 1 分钟

Vim /etc/ssh/sshd_config

找到port 22 在下面加上要更改的端口

在可以通过新端口访问前，不要关闭现有连接！！！！

默认情况下，SELinux 只允许端口 22 用于 SSH，现在需要做的是通过 SELinux 启用新创建的端口
semanage port -a -t ssh_port_t -p tcp 10252
如果要移除该端口，使用 -d参数
semanage port -d -t ssh_port_t -p tcp 10252

如果未安装semanage，执行yum -y install policycoreutils-python

允许新端口通过防火墙
firewall-cmd –permanent –zone=public –add-port=10252/tcp

重启防火墙
firewall-cmd –reload

重启防火墙服务
systemctl restart sshd.service

查看新端口是否生效
systemctl status sshd.service

使用新端口进行连接    

Centos7 防火墙相关操作
2021年05月24日 89 字 大概 1 分钟

查看zone名称：
firewall-cmd –get-active-zones

开启端口（永久）
firewall-cmd –zone=public –add-port=6379/tcp –permanent

重启防火墙
firewall-cmd –reload

查看端口号是否开启
firewall-cmd –query-port=6379/tcp

查看防火墙状态
firewall-cmd –state

停止防火墙
systemctl stop firewalld.service

禁止防火墙开机自启
systemctl disable firewalld.service

启动防火墙
systemctl start firewalld.service

Git 项目上传到 svn中
2021年05月24日 172 字 大概 1 分钟
  1. 创建svn库
    svn co http://svn.example.com/foo
    cd myproj
    svn mkdir trunk
    svn commit -m’Created trunk directory’
    或者直接在现有的repository上创建目录
    svn mkdir –parents http://url/dir_name –message “messages”
  2. 设置svn远程库
    clone git项目
    git clone ~git/repositories/foo/mainline.git ~git/repositories/svn-mirror/foo
    cd ~git/repositories/svn-mirror/foo
    在.git/config中添加svn-remote
    [svn-remote “svn”]
    url = http://svn.example.com/foo/trunk
    fetch = :refs/remotes/git-svn
  3. 提交项目到svn库
    从空的svn远程库中做初始化fetch,并将其作为一个新分支checkout
    git svn fetch svn
    git checkout -b svn git-svn
    将master分支merge进svn分支并提交到svn库merge时发生fatal: refusing to merge unrelated histories错误, 可加–allow-unrelated-histories
    git merge master
    git svn dcommit
  4. rebase到svn分支以便从master版本推送到svn库
    git checkout master
    git rebase svn
    git branch -d svn

如何将github上的public项目fork为private项目
2021年05月24日 59 字 大概 1 分钟

1.先在GitHub上创建一个private项目
2.将public项目clone到本地
git clone –bare https://github.com/exampleuser/public-repo.git
3.进入到刚刚clone的项目里
cd public-repo.git
4.push到刚刚创建的private项目中
git push –mirror https://github.com/yourname/private-repo.git
