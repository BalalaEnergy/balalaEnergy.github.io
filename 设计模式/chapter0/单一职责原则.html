Single Responsibility Principle：单一职责原则

Single Responsibility Principle：单一职责原则

通常来说，一个类只负责一种职责。假设一个类中有多种职责同时存在，那么修改其中一种职责时，可能回对其他职责造成不必要的影响。

e.g.

public class Taxi {
    /**
     * 启动引擎
     */
    public void start() {
        System.out.println("出租车发动");
    }
    /**
     * 熄火
     */
    public void stop() {
        System.out.println("出租车停止");
    }
    /**
     * 前进
     */
    public void run() {
        System.out.println("出租车前进");
    }
    /**
     * 乘客上车
     */
    public void passengerUp(){
        System.out.println("乘客上车");
        start();
    }
    /**
     * 乘客下车
     */
    public void passengerDown(){
        stop();
        System.out.println("乘客下车");
    }
}

这个例子中，Car这个类包含了汽车的职责和接送乘客的职责，如果修改了其中一项职责，相当于同时修改了另一项职责，可能会对引用它的地方造成一定的影响。

public class Taxi {
    /**
     * 启动引擎
     */
    public void start() {
        System.out.println("出租车发动");
    }
    /**
     * 熄火
     */
    public void stop() {
        System.out.println("出租车停止");
    }
    /**
     * 前进
     */
    public void run() {
        System.out.println("出租车前进");
    }
}

public class TaxiWork {

    private Taxi taxi;

    /**
     * 乘客上车
     */
    public void passengerUp(){
        System.out.println("乘客上车");
        taxi.start();
    }
    /**
     * 乘客下车
     */
    public void passengerDown(){
        taxi.stop();
        System.out.println("乘客下车");
    }
}

为什么遵守

  • 降低类的复杂度，一个类只负责一项职责
  • 提高类的可读性，可维护性
  • 降低变更引起的风险

ps

通常情况下，我们应当遵守单一职责原则，只有逻辑足够简单，才可以在代码级违反单一职责原则；只有类中方法数量足够少，可以在方法级别保持单一职责原则
