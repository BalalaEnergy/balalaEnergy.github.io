Open Closed Principle：开闭原则

Open Closed Principle：开闭原则

一个类应该对外扩展开放，对内修改封闭。当程序需要变化时，应该通过扩展的方式进行改变，而不是修改已有代码。

e.g.

public class Driver {
    public void driver(Car car) {
        if ("Taxi".equals(car.getType())) {
            System.out.println("driver taxi");
        } else if ("Didi".equals(car.getType())) {
            System.out.println("driver didi");
        }
    }
}

public class Car {
    private String type;

    public String getType() {
        return type;
    }
}

如果司机需要驾驶顺风车，那么还需要修改driver方法，可能会导致已有代码不稳定。 我们需要给Car提供可扩展的能力，新加功能时通过扩展的方式实现它。

public class Driver {
    public void driver(Car car) {
        car.driver();
    }
}

public abstract class Car {
    private String type;

    public String getType() {
        return type;
    }

    public abstract void driver();
}

public class Taxi extends Car {
    @java.lang.Override
    public void driver() {
        System.out.println("driver taxi");
    }
}

public class Didi extends Car {
    @java.lang.Override
    public void driver() {
        System.out.println("driver didi");
    }
}

public class RideSharing extends Car {
    @java.lang.Override
    public void driver() {
        System.out.println("driver ride sharing");
    }
}

为什么遵守

  • 可以方便的扩展功能而不用修改已有代码，提高系统的可维护性。
