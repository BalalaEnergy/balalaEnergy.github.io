Composite/Aggregate Reuse Principle：聚合复用原则

Composite/Aggregate Reuse Principle：组合/聚合复用原则

如果使用继承的话，父类中增加了方法，子类对象同时也会增加该方法。如果子类对象中恰巧之前也写过这个方法，可能会对开发者造成一定的影响；其次，父类中增加的方法有些子类可能并不会用到。

e.g.

// 鸟类行为
public class BirdBehavior {
    public void eat() {
        System.out.println("鸟类觅食");
    }
}

public class Sparrow extends BirdBehavior {
    // 麻雀
}

public class Ostrich extends BirdBehavior {
    // 鸵鸟
}

假设现在要给BirdBehavior类中增加fly()方法，这会导致鸵鸟类中继承了并不会使用到的方法。如果继承关系特别复杂时，会增加维护的难度。

使用组合的方式：

// 鸟类行为
public class BirdBehavior {
    public void eat() {
        System.out.println("鸟类觅食");
    }

    public void fly() {
        System.out.println("鸟飞走了");
    }
}

public class Sparrow {
    private BirdBehavior birdBehavior;
    // 麻雀
}

public class Ostrich {
    private BirdBehavior birdBehavior;
    // 鸵鸟
}

组合的方式将变化放到了需要变化的类中，并不会对其他的类造成影响。

为什么遵守

  • 修改各个复用到的类变得容易，不用担心会影响到其子类。
  • 可以动态的替换各个复用类。
  • 各个复用类各司其职，在其他地方一样也可以使用，提高了代码复用。
