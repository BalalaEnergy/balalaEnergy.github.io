Liskov Substitution Principle：里氏替换原则

Liskov Substitution Principle：里氏替换原则

子类对象可以在程序中代替其基类对象。

e.g.

public class A {
    public void add(int i1, int i2) {
        System.out.println(i1 + i2);
    }
}

public class B extends A {
    @java.lang.Override
    public void add(int i1, int i2) {
        System.out.println(i1 + i2 + 100);
    }
}

public class Use {
    public static void main(String[] args) {
        A a = new A();
        a.add(1,2);
        B b = new B();
        b.add(1,2);
    }
}

调用B类的add方法和A类的add方法结果不同，可能会对开发者造成预料之外的影响。因为开发的时候可能不会点到子类中去看一下具体的实现方式。

里氏替换原则的实践可以归纳为以下四点：

  • 子类必须实现父类的抽象方法，但不得重写（覆盖）父类的非抽象（已实现）方法。
  • 子类中可以增加自己特有的方法。
  • 当子类覆盖或实现父类的方法时，方法的前置条件（即方法的形参）要比父类方法的输入参数更宽松。
  • 当子类的方法实现父类的抽象方法时，方法的后置条件（即方法的返回值）要比父类更严格。

为什么遵守

  • 规范了父类与子类之间安全的继承，继承是安全的，代码的复用才是安全的。
  • 继承也提高了代码可维护性，方便修改父类的公共方法和子类的特定方法。
