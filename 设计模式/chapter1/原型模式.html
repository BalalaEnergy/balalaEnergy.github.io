原型模式

原型模式 Prototype Pattern

何时使用

  • 当一个对象的构建代价过高时。例如某个对象里面的数据需要访问数据库才能拿到，而我们却要多次构建这样的对象。
  • 当构建的多个对象，均需要处于某种原始状态时，就可以先构建一个拥有此状态的原型对象，其他对象基于原型对象来修改。

如何使用

原型模式必须提供一个可以克隆自己的方法。可以通过实现Java的Cloneable接口，或者自己定义一个接口。 使用Java的Cloneable接口，返回值是Object，如果需要准确的返回值类型，推荐自己定义接口。

public interface ReportPrototype {
    ReportPrototype copy();
}

接口定义完成后，我们还需要提供一个原型类，供其他人copy。

public class Report implements ReportPrototype {
    List<Data> data;

    public void loadData() {
        // 假设这一步需要2秒才能加载完成
        Thread.sleep(2000);
        this.data = new ArrayList<>();
    }

    @java.lang.Override
    public ReportPrototype copy() {
        // 这一步可以是深拷贝，也可以是前拷贝，根据业务需求来决定。
        return new ArrayList<>(data);
    }
}

原型类创建完成后，就可以将其提供给其他人来使用了。

public class PrototypeUse {
    public static void main(String[] args) {
        ReportPrototype report = new Report();
        report.loadData();
        //使用该数据
        System.out.println(report);
        //另一个场景也需要使用report
        ReportPrototype reportCopy = report.copy();
        System.out.println(reportCopy);
    }
}
