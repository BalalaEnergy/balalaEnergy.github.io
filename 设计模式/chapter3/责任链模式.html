责任链模式

责任链模式

使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系。将这些对象连成一条链，并沿着这条链传递该请求，直到有一个对象处理它为止。

审批就是一种常见的责任链模式。根据不同的申请决定不同的处理方式，但是调用方的入口一直都应该只有一个。

  • 部门leader 可以审批1天以下的假
  • 部门总监 可以审批3天以下的假
  • CTO 可以审批3天以上的假

想要使用责任链模式，我们首先得抽象出一个接口。true代表审批通过，false代表不通过，null代表没权限

public interface Handler {
    Boolean process(int days);
}

不同的角色实现不同的类

public class ManagerHandler implements Handler {
    public Boolean process(int days) {
        if (days > 1) {
            return null;
        }
        // 模拟审批逻辑
        if (ok) {
            return true;
        }
        return false;
    }
}

public class DirectorHandler implements Handler {
    public Boolean process(int days) {
        if (days <= 3) {
            return null;
        }
        // 模拟审批逻辑
        if (ok) {
            return true;
        }
        return false;
    }
}

public class CtoHandler implements Handler {
    public Boolean process(int days) {
        // 模拟审批逻辑
        if (ok) {
            return true;
        }
        return false;
    }
}

创建好实现类后，我们需要将这些类链起来

public class HandlerChain {
    List<Handler> chain = new ArrayList<Handler>();

    public void addHandler(Handler handler) {
        chain.add(handler);
    }

    public boolean approve(int days) {
        for (Handler handler : chain) {
            Boolean status = chain.process(days);
            if (status != null) {
                return status;
            }
        }
    }
}

之后我们便可以使用它了

public class Approve {
    public static void main(String[] args) {
        HandlerChain chain = new HandlerChain();
        chain.addHandler(new ManagerHandler());
        chain.addHandler(new DirectorHandler());
        chain.addHandler(new CtoHandler());

        chain.approve(1);
    }
}

上述的例子中，处理器的顺序很重要，如果顺序不对，得到的结果可能会出现不一致的情况。

除此之外，责任链还有很多种变种，比如Interceptor、Filter。他们的特点是每一个实现类都会有处理请求的机会，具体的写法和上述例子差不多。比如mybatis就可以定义多个Interceptor。
